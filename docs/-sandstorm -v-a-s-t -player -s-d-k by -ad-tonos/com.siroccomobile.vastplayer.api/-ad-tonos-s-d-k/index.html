Shared object for VAST advertisements handling. This is the place where developer should start reading. AdTonosSDK is an object containing a set of methods for generating and playing VAST advertisements. See Quick Guide for more information.

Installation

Add the following code to the application manifest between the application tags:

<activity android:name="com.siroccomobile.linkgenerator.adtonos.ui.AdTonosActivity" android:theme="@style/AppTheme.Transparent" />
Content copied to clipboard

Usage:

  1. Call initialize at initialization time of the app.

  2. Call start

  3. Call createBuilder and fill key with AdTonosVastUrlBuilder.setAdTonosKey

  4. Call requestForAds passing builder as param. Make it on UI thread.

  5. Call playAd when ad is loaded.

  6. Repeat step 4 and 5 after ads finished playing

  7. When application is being destroyed call dispose method to free all resources

pauseAd
Link copied to clipboard
open override fun pauseAd(): Boolean
Content copied to clipboard

Method that should be called whenever the application loses foreground. MUST BE CALLED ON UI THREAD. Its task is to perform a pause on the currently playing material. If the method succeeds true value is returned and OnAdTonosVastAdPaused method of AdTonosVastPlayerCallback is called. If the pause fails while advertising material is available, the callback OnAdTonosVastError (PauseAdError) from AdTonosVastPlayerCallback is called. In the case where the pause is due to a change in the application foreground and does not affect the playback of the advertisement, only the value false will be returned.

requestForAds
Link copied to clipboard
open override fun requestForAds(context: Context, builder: AdTonosVastUrlBuilder): AdTonosAdRequestResult
Content copied to clipboard

The method call performs a query for the ads and then downloads them. MUST BE CALLED ON UI THREAD. When the process is complete and ready, the onAdTonosVastAdsLoaded method is called. From this moment on it is possible to call playAd. The method, depending on the current state, returns the appropriate value of the AdTonosAdRequestResult enum informing what is currently happening. After calling the clear method will be destroyed and their instances should not be used again. In order to be able to play the next advertisement, the playback of the previous ones must be completed. See the example in the Quick Guide to see how the whole process can be combined.
